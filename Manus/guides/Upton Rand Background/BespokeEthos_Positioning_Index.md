## SECTION 12: GOOGLE CLOUD / VERTEX AI STRATEGY
*From The Vertex Architect Document*

### 12.1 Core Strategic Framework
**The Problem Being Solved:** Analysis paralysis in website development ("fiddling far too long")
**The Solution:** Agentic AI workflow that mandates execution speed

**Key Constraints (Non-Negotiable):**
- Static site output only (Vercel/serverless compatible)
- No virtual servers
- Low-maintenance architecture
- Simple web component/API integration for chatbot

---

### 12.2 The Founder-as-Architect Model
**Role Transition:** From coder/executor → orchestrator/manager of AI agents
- Primary job becomes strategic direction and verification
- Agents generate "Artifacts" (tangible deliverables) for verification
- Feedback happens on Artifacts without stopping execution

---

### 12.3 System Prompt Architecture ("The Constitution")
**Purpose:** Long-form foundational document encoding:
- Ethical constraints
- Brand identity
- Operational protocols
- Mission adherence

**Required Elements:**
| Dimension | Data Points | Desired Behavior |
|-----------|-------------|------------------|
| Core Mission | NGLCC-certified, LGBTQ-focused, equitable AI adoption | Prioritize accessibility, inclusive language, cost-effectiveness |
| Economic Mandate | AI correlates with 82% workforce growth | Frame solutions as workforce expansion accelerators |
| Tone/Voice | Authoritative, supportive, inclusive, visionary | Prohibit corporate jargon, maintain empowerment focus |

---

### 12.4 Website Deployment Prompts (Phase I)

**Prompt I: Architectural Planner**
- Convert Figma → Framer → static HTML
- Mobile-first, accessibility-compliant
- Artifact: Implementation Plan (PDF)

**Prompt II: Component Generation**
- Use Framer AI tools (Wireframer, Workshop)
- Cookie banner, multi-language toggle, pricing table
- Artifact: Component List + Screenshots

**Prompt III: Quality Assurance**
- Cross-browser testing (Chrome, Safari, Mobile)
- Artifact: Screenshots (3 breakpoints) + Browser Recording

---

### 12.5 Chatbot Architecture Prompts (Phase II)

**Prompt I: Persona Definition**
- "Growth Architect" persona
- Knowledgeable, supportive, inclusive guide
- Store all conversation history for rapport building
- Artifact: Persona Document (PDF)

**Prompt II: Narrative Architecture**
- Use narrative theory for engagement
- Encoder/decoder architecture for personalization
- Simple serverless embed code
- Artifact: Conversation Flow Map + Integration Plan

**Prompt III: Data Strategy**
- Prioritize long-form text over fragments
- Ingest: founder interviews, mission docs, white papers
- Artifact: Data Ingestion Plan (Schema)

---

### 12.6 MLOps and Governance

**Evaluation Metrics:**
1. Brand Tone Adherence
2. Successful Solution Delivery Rate
3. Customer Satisfaction Scores

**Feedback Loop:** Direct annotation on Artifacts → agent incorporates without stopping

**Final Deployment Checklist:**
- [ ] NGLCC ethical constraints met
