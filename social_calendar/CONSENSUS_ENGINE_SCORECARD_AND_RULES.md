## Executive Summary

The Consensus Engine is a competitive AI research framework that pits multiple AI models against each other to verify the accuracy of community event data. By leveraging AI competition, we ensure higher accuracy, transparency, and community trust in event information.

**Mission:** When AI models compete, communities benefit through improved accuracy and accountability.

## Scoring System

### Point Allocation

| Action | Points | Notes |
|--------|--------|-------|
| Verified Event | +10 | Confirmed via multiple sources |
| Unique Finding | +20 | Event found only by this model |
| Faulty Event Detection | +5 | Correctly identified error |
| Faulty Event (Model's Error) | -10 | Model provided incorrect data |
| Recurring Error Pattern | -15 | Systematic mistake across events |
| Minimal Findings | -6 | Less than 3 events identified |

### Verification Criteria

An event is marked **VERIFIED** when:
- ✅ Confirmed on official venue website or social media
- ✅ Multiple independent sources corroborate details
- ✅ Event date, time, and location are accurate
- ✅ Event is accessible to the LGBTQ+ community

An event is marked **FAULTY** when:
- ❌ Date is incorrect or event is canceled
- ❌ Time is wrong (off by 30+ minutes)
- ❌ Location is incorrect or venue is closed
- ❌ Event does not exist or is AI hallucination

## Rules for AI Models

### Rule 1: Data Accuracy
- All event information must be verified from primary sources
- Models must cite their sources for each event
- Unverified events must be marked as "UNVERIFIED"

### Rule 2: Community Focus
- Priority given to events serving trans, lesbian, and marginalized communities
- Events must be accessible and welcoming to LGBTQ+ participants
- Cultural sensitivity is mandatory

### Rule 3: Completeness
- Models must identify recurring events (weekly, monthly, daily)
- Models must include event details: date, time, location, price
- Models must provide descriptions for community events

### Rule 4: Transparency
- All faulty events must include evidence of the error
- Models must explain their research methodology
- Conflicts between models must be documented

### Rule 5: Timeliness
- Events must be current and relevant (within 6-month window)
- Quarterly updates required
- Past events must be removed

### Rule 6: Inclusivity
- Events must represent diverse LGBTQ+ communities
- Nightlife, community, recovery, culture, and gaming categories required
- Trans and lesbian events must be prioritized

## Methodology Notes

### Research Process
1. **Primary Source Verification:** Official websites, social media, venue confirmation
2. **Secondary Source Validation:** News articles, community calendars, event listings
3. **Community Feedback:** Direct outreach to LGBTQ+ community members
4. **Cross-Reference:** Multiple models compared for accuracy

### Data Quality Standards
- ✅ All dates verified against official sources
- ✅ Times confirmed within 15-minute accuracy window
- ✅ Locations verified as active and accessible
- ✅ Descriptions written by community members when possible

### Limitations & Caveats
- Some events may be private or by-invitation-only
- Recurring events subject to cancellation or schedule changes
- Quarterly updates may miss last-minute event announcements
- Virtual events not included in current version
